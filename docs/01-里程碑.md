## 端到端总验收剧本（M9 用，10 步全过 = MVP 成立）

1. 新用户扫码 → 授权登录 + 手机号
2. 看到两段式同意，健康数据**非默认勾选**
3. 轻建档 1 屏完成进首页
4. **关蓝牙**称重 → 手动 20 秒录入；改一笔为拍秤照
5. 拍一餐真实饭菜 → AI 给菜品+能量 → 修正一项 → 入账
6. 拍一张零食营养成分表 → OCR 读出能量 → 填食用量 → 入账
7. 今日汇总 → 一键转发到测试微信群 → 群内卡片可点回小程序
8. 用户把数据共享给两位服务者 → 服务者甲打开"我服务的人"，看到该用户当日记录并写一条服务备注
9. 用户移除服务者乙 → 乙立即失访、列表消失
10. 公司后台看到该用户全部记录与共享关系 → 改一次服务老师标签（留日志）→ 导出 CSV

## 合规自查清单（M9 提审前逐条打勾）

1. ☐ S2 两段式同意：健康数据独立、非默认勾选，拒绝后记录功能锁定
2. ☐ 用户可删单条记录、可撤回同意、可注销且餐照连删
3. ☐ 全部用户可见文案 + 分享卡片 + 图片内文字过轻体词表（CI 扫描 + 人工抽查）
4. ☐ AI 输出过滤层生效（构造诱导图：药品/体检单照片不产生医疗表述）
5. ☐ 餐照/秤照私有桶、签名 URL 短时效、非授权不可访问
6. ☐ 后台仅 admin/viewer 两档账号，改归属留日志
7. ☐ 隐私政策 + 同意文案经法务确认
8. ☐ VTrump 数据流向已确认并按结论处理
9. ☐ 共享：移除服务者后立即失访（接口 403）；邀请码 24h/单次失效生效；服务者无法导出、无法改原始数据
10. ☐ 阶段记录：体型照/舌诊照/体检报告全私有桶+签名 URL；visible_to_advisors 双闸生效；**全系统无任何对报告/舌诊的自动解读输出**
11. ☐ 图片下载：上传端 EXIF（含 GPS）剥离生效；后台单张/批量下载全部写入 media_download_logs 审计；批量导出仅 admin 角色；导出 zip 链接 24h 失效
