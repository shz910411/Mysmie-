# 00 · 需求规格（17 屏 + 8 后台模块）

> 项目宪法见 [../CLAUDE.md](../CLAUDE.md)。本文件为冻结需求规格，遇与 PM 窗工作记录冲突以本文件为准。

## 一、主线一句话

进来 → 1 屏建档 → 每天"称一下 + 拍一拍" → AI 告诉你吃了什么、大概多少能量 → 当日汇总 → 一键转发到微信群。

## 二、用户端 17 屏

### 入口段（S1-S3）

| 屏 | 名称 | 关键点 | 成功标准 |
|---|---|---|---|
| S1 | 登录 | 微信授权 + 手机号原生授权（不传明文）| 新用户 ≤3 步进首页；手机号入库 |
| S2 | 双层同意 | ①通用隐私 ②**健康数据单独勾选**（非默认、不捆绑）| 未勾健康同意则记录功能锁定；同意带版本入库 |
| S3 | 轻建档 | **仅 3 项**：性别 / 年龄 / 身高（体重首次称重产生）| 1 屏完成；目标体重在"我的"里选填 |

### 主线段（S4-S8）⭐ 项目心脏

| 屏 | 名称 | 关键点 | 成功标准 |
|---|---|---|---|
| S4 | 主页 = 称重 | 顶部 logo + 日期；称重主卡（今晨体重大字 / 较昨日 / 来源）；CTA「开始称重」；下方今日摄入 + 快捷行 + 我服务的人 | 打开即称重页；任何记录 5 秒内反映 |
| S5 | 称重流程 | 蓝牙连接动画 → 读数；**15 秒超时**自动切手动录入 + 拍秤照 | 断网/无蓝牙/无秤照样 20 秒内记录；source(ble/manual/photo) 正确入库 |
| S5b | 数据对比页 | **14 维度全展示**（体重/BMI/体脂/脂肪/皮下/内脏/水分/骨骼肌/肌肉/骨量/基础代谢/蛋白质/体年龄/健康评分）+ 鼓励语轮换 + 中性数值解读 | 蓝牙称重后 1 步看到全维度；任意两次可对比；手动录入降级为简版 |
| S6 ⭐ | 拍照吃饭 + AI 解析 | 选餐次 → 拍照 → AI 返回菜名+份量+kcal（**≤8 秒**）→ 用户可改份量/删/加；**三段反馈**：211 搭配 / 份量节奏 / 营养缺口 | 一餐 ≤30 秒入账；AI 失败照片不丢；能量标"估算" |
| S7 | 配料表 OCR | 拍营养成分表 → 识别 kJ/kcal → 用户填食用量 → 入账 | 国标 GB 28050 识别率 ≥80%；失败可手填 |
| S8 | 今日汇总 + 分享 | 全维度日报 + 自动小结（**规则生成**中性描述，非 AI）；分享卡片**两路**：原生 `open-type=share` + 长图保存 | 2 步内转发到群；卡片**零**"瘦/减/燃脂"字样 |

### 个人空间段（S9-S11）

| 屏 | 名称 | 关键点 | 成功标准 |
|---|---|---|---|
| S9 | 我的 | 资料 / 历史流水 / 数据共享入口 / 健康授权管理 / 删除注销 / 联系客服 | 能自助删单条；能发起注销 |
| S10 | 数据共享管理 | 生成邀请（卡片+二维码，**24h 单次有效**）→ 微信发服务者；共享名单可一键移除 | 共享给 ≥2 人同时生效；移除即时失访；建立/移除留日志 |
| S11 | 服务视图 | 同一小程序内的另一身份；首页"我服务的人"入口；列表 + 详情（**用户当日汇总的只读镜像**）+ 归档/备注 | ≤2 步打开某用户当日记录；被移除立即失访；备注留痕 |

### 进阶段（S12-S17）

| 屏 | 名称 | 关键点 | 成功标准 |
|---|---|---|---|
| S12 | 健康调研表 | 6 组 40 题选择题（目的/基础情况/症状/生活习惯/其他）；**基础情况细化 8 组系统**（敏感）| 未填者首页常驻入口；提交后档案完整度 100% |
| S13 | 体重曲线 | 累计变化大数字 + 全周期折线 + 起始/当前/距目标/周均 | 与称重记录一致 |
| S14 | 健康数据授权管理 | 三项授权状态 + 共享名单 + **撤回授权** + **注销（餐照连删）**| 撤回后记录功能锁定 |
| S15 | 阶段记录 | 体型照三面（正/侧/背）+ 围度（自动算腰臀比）+ 体检报告 + 舌诊照；**多期制用户自主**| 基线 ≤2 分钟；腰臀比正确；可见性开关即时 |
| S16 | 阶段对比 | 任意两期对比 + AI 阶段小结（**仅基于结构化数据**，不进图片/报告）+ 老师人工分析 | 服务者在双闸关闭时不可见照片 |
| S17 | 开启之礼 · 新手任务 | 5 项（手机号/调研/体型照/报告/舌诊）→ 4/5 提醒 → 5/5 庆祝弹窗 → 填收货 → 寄**轻体随手杯 250ml**| 进度即时；庆祝仅一次；地址必填 |

## 三、通知与提醒系统（贯穿全屏）

10 类通知 + `notice.target` 字段驱动跳转 + 频控每日同类 ≤1 条 + 通知中心 🔔：
- 新一期营邀请、健康调研未填、今日未称重、今日未记录餐食、老师的话已送达、阶段记录提醒（每 2 周）、老师阶段分析出炉、共享请求/公司指派、运营内容、结营成绩单（v1.1）

成功标准：后台发送→用户下次打开必弹；点通知按 target 直达对应页；接受/拒绝状态回传后台；频控生效。

## 四、饮水/排便（选填，无完成度判定）

- **可加可减**步进器：饮水默认 1 杯≈250ml（"我的"里可调）；排便次数 ± + 状态三选（顺畅/一般/不太顺）
- "想记就记，没有必填/缺卡概念"——v2 是记录工具不是监工

## 五、公司后台 8 模块（唯一管理端，串行开发）

| # | 模块 | 关键功能 | 成功标准 |
|---|---|---|---|
| 1 | 总览 | KPI（注册/活跃/日均打卡/警示数）+ 未打卡名单一键提醒 + 动态流 | 一屏看全公司状态 |
| 2 | 打卡总表 | 全员矩阵：晨重/三餐胶囊（含 kcal）/饮水/排便/完成度条；日期切换 + 只看未完成；点 ✓ 看打卡内容 | 千级用户秒开 |
| 3 | 日报汇总 | 四率 KPI + 今日亮点 + 需关注 + 老师服务情况（**服务质量口径，不作计酬**），按日切换 | 运营拿来排周报 |
| 4 | 用户管理 | 搜筛（手机号/服务老师/分组/注册日期）→ **360° 档案六标签**（概况/打卡/调研/阶段/备注/归属与操作）| 一屏看全一个用户 |
| 5 | 警示中心 | 规则触发分级：**连续 2 天未记录·中→4 天·高公司介入** / 单日波动>1kg / 调研超 48h；一键提醒/标记处理留痕；规则可配置 | 警示触发即时；处理留痕 |
| 6 | 提醒中心 | 圈人群发 → 站内弹窗（频控）| 圈人精准；频控生效 |
| 7 | 营期分析 | 每日完整打卡率趋势 + 体重变化分布 + 211 达标率 + 最常缺项；**只作运营参考不输出个体健康结论** | 运营可用 |
| 8 | 共享与记录 | 共享关系审计 + 发送记录 | 谁在看谁数据一目了然可审计 |

**导出**：打卡总表 / 用户列表 / 日报 / 单用户记录均可一键 CSV（**不含饮食照片**）；批量导出走异步打包+24h 失效签名+`media_download_logs` 审计。

**账号权限**：仅 admin（可改归属）/ viewer 两档。

## 六、关键已拍板决策（不再回头问）

| 决策 | 结论 |
|---|---|
| 产品定位 | 纯使用工具，无支付无团队，运营在微信 |
| 归属 | 单层"服务老师"标签（后台可改不可见用户端）|
| 九洲福对齐 | v1 不打通，只收手机号，后台离线名单比对 |
| 建档深度 | 轻建档：性别+年龄+身高（体重首次称重产生）|
| AI 视觉 | **直接用 Chester 提供的旧版 API**，不再选型 |
| 后台可见性 | 公司全量可见、可调归属 |
| 数据共享 | 用户主动共享给多名，扁平无层级，可随时移除 |
| 视觉规范 | 对齐 VI 1.0.0，主色 `#0D8456`，HarmonyOS Sans + Didot 衬线数字 |
| 称重为主页 | "大家第一反应是来称重的" |
| 餐食 AI 反馈 | 三段式：211 搭配 / 份量节奏 / 营养缺口（非医疗）|
| 开启之礼 | 5 项新手任务 → 寄轻体随手杯 250ml（与饮水默认杯量对齐）|
| 后端部署 | 自建于阿里云/华为云（与九洲福同云）|
| 体重秤 | vtble 插件 P1，**手动/拍秤 P0** 永远不可被卡死 |
| 用户唯一性 | 登录身份=微信 openid；业务身份=手机号（DB 唯一约束）|

## 七、明确不做（防需求回潮）

- ❌ 小程序内排行榜 / 奖励 / 勋章
- ❌ 团队层级展示 / 服务人数排行
- ❌ 独立经销商后台
- ❌ 自动推送
- ❌ 医疗化饮食评价（疾病关联 / 效果承诺 / 否定式 / 营养处方）
- ❌ 营期系统 / 打卡完成度判定 / 补卡规则（→ 微信群人肉）
- ❌ 支付 / 商城 / 分润（→ 九洲福的事）
- ❌ AI 营养建议（合规红线，只做中性记录）
- ❌ 报告 / 舌诊照的 AI 自动解读（人工出具）
- ❌ 自动发群（用户主动点 share 才发）
