# 03 · UI 蓝本（前端实现基准）

> 来源：Chester 已调校确认的高保真雏形 `design-reference/user-prototype.html`（用户端 22 屏，v0.7 编辑式重设计 2026-06-11）+ `design-reference/admin-prototype.html`（后台 v2）。
> **规则**：所有前端 Issue 照本蓝本 1:1 还原视觉/尺寸；代码用小程序 wxml/wxss 从零写，不搬旧逻辑。雏形是"皮肤蓝本"，不是代码来源。

## ⚠️ 头号注意：CSS 三层叠加，只实现最终 EDITORIAL 值

雏形 `<style>` 是四段按顺序覆盖，后写的赢。Dev 做 wxss **只实现最终生效值**，别照抄基础层：

| 层 | 作用 |
|---|---|
| 基础层 | 原始 VI 令牌、圆角大、卡片有边框、tab 带 emoji |
| LUX 层 | Didot 衬线、圆角收小、字距加宽 |
| **EDITORIAL v0.7（最终生效）** | **去卡片化（→发丝分隔线）、去 emoji（tab 纯文字）、直角按钮、下划线输入框、超大 Didot 数字** |
| cmp-table v2 | 对比表年报式右对齐等宽数字 |

例：`.btn` 基础层 `border-radius:14px` 翠绿 → 最终覆盖为 `border-radius:2px` 墨绿底 `#233710` `letter-spacing:.32em`。**要 2px 直角墨绿，不是 14px 圆角翠绿。**

### 2.3 间距/圆角
- 屏内边距 `.screen`：`50px 26px 92px`（移动顶 14px+安全区/底 88px+安全区）
- 卡片（去边框分区）：`padding:26px 0`，无左右内边距，`border-bottom:1px solid --line2` 分隔，末块去线
- 左强调条卡（AI 反馈/小结）：`border-left:2px solid --green; padding-left:16px`
- **圆角基本归零**：按钮 2px、输入框 0（下划线）、卡片 0、tab 0、日历格/胶囊/segment 2px、头像 50%
- **阴影**：UI 内部几乎无阴影（编辑式扁平）；手机框投影是网页外壳，小程序不要

## 3. 核心组件规范（Chester 调校数值，逐一给准）

### 3.1 按钮
| 类型 | class | 尺寸 | 圆角 | 字号/字距 | 颜色 |
|---|---|---|---|---|---|
| 主按钮 | `.btn` | `width:100%;padding:17px` | 2px | 13px/.32em | 底 `--forest`、字 #fff、500 |
| 主按钮·白底(深底上) | `.btn.onGreen` | 同 | 2px | 同 | 底 #fff、字 --forest |
| 次按钮(文字链) | `.btn.ghost` | 无边框无底 | — | 13px/.12em | 字 `--ink2`、下划线 offset 5px |
| 小按钮(行内) | `.btn.small` | `padding:10px 18px` | 2px | 12px/.14em | 墨绿 |
| 禁用 | `:disabled` | — | — | — | 底 #D4D1C5 |
| 纯文字链 | `.linklike` | — | — | 13px/.04em | #ink2 下划线 offset 4px |

**次操作不是描边按钮，是"下划线文字链"**（编辑式核心，页面大量"去看看/记/查看"是文字链）。小程序需自加 `hover-class`/`:active` 做 8% 加深反馈。

### 3.2 卡片/分区
**不用卡片堆叠，用发丝线节奏+大留白**：`background:transparent;border:0;border-radius:0;padding:26px 0;border-bottom:1px solid #E8E5DB`。小程序用 `view`+`border-bottom`，**别加背景色/圆角/投影**。例外：强调块 `border-left:2px solid #0D8456`+微底色（如系统提醒 `#FDF8F4`）。

### 3.3 tabbar ⚠️（雏形是 3 tab 纯文字无图标）
- 雏形 `const TABS=['home','summary','mine']` = **称重/汇总/我的 3 个 tab**，"打卡"并入称重主页（主页有"开始称重"+"拍照打卡"两入口）。**⚠️ 与文档 00「4 tab」冲突，待 PM/Chester 裁决**。
- 高度 64px（移动 60px+安全区），底 fixed，顶 `border-top:1px solid #E8E5DB`，纸色底
- `.ico{display:none}` emoji 全隐藏，只留文字 `font-size:12px;letter-spacing:.34em`，未选 #B9B6A9，选中 `--ink2`+600
- 原生 `tabBar` 不支持纯文字+超宽字距 → 需 `custom-tab-bar` 自定义组件，或退用极简单色图标

### 3.4 数据对比表 cmp-table（年报式，S5b 14 行 + S16 4 行复用）
- 四列 fixed：指标(31%)/昨值(24%)/今值(24%)/变化(19%)
- 表头 th：9.5px/.18em/#A9A698/右对齐（首列左）/`border-bottom:1px solid --ink2`
- 指标列：主名 13.5px 加粗 + 下方参考说明 9.5px #B5B2A4（如"目标 52.0kg""女性参考 <25%"）
- 数字列：**Didot 17px + tabular-nums 等宽 + 右对齐 + nowrap**；昨值灰 #A9A698、今值 `.today` #1B1C17；状态标 `.st`（good 绿/warn #A8814A）9px 在数字下
- 变化列：Didot 13.5px 等宽，`↓/↑` 前缀，`—`无变化 #C9C6B8；**中性色不上红绿**（避免"下降=好"误导，好坏交状态列）
- 14 项：体重/健康评分/BMI/体脂率/脂肪/皮下脂肪率/内脏脂肪等级/水分/骨骼肌/肌肉量/骨量/基础代谢/蛋白质/体年龄（对齐 VTrump 全维度）

### 3.5 其它高频组件
- 输入框：`border:0;border-bottom:1px solid --ink2;background:transparent;padding:10px 2px;font-family:Didot;font-size:19px`，聚焦 border 变绿
- 页眉 brandbar：左 logo(`img` h34px) + 右文字块右对齐（`MYSMIE` 14px/.38em/#ink2 + `迈思美` 9px/.5em/#A9A698）；主页 S4 用居中变体
- 步进器 stepper：圆按钮 46×46 圆形 1.5px 边，中间数字 32px，间距 24px
- 份量胶囊 portion：圆角 2px，未选 1px 灰边字 #8A8A80，选中 `--forest` 实底白字
- 打卡日历 cal：grid 7 列 gap6px，格高 32px 圆角 0，full 绿底白字/part 浅绿 #D8E6CB/未记录 #F1EFE5
- 趋势条 spark：flex 底对齐 height54px gap10px，柱 #EFEDE3 圆角 0，高亮柱绿
- 头像 avatar：42×42 圆，浅灰底 #EFEDE3，**姓氏单字**(Didot)代头像
- 照片格 photo-slot：`aspect-ratio:3/4`（报告 3/2），虚线边 1px dashed #C9C6B8 底 #F7F6EE，已传变实线绿边
- 丝带 ribbon：inline SVG 贝塞尔曲线（登录页/对比 hero 底/分享卡底）→ 小程序导出透明 PNG 或 canvas

## 4. 主线屏布局（从上到下，M2/M3 重点）

**S4 称重主页 `home`**：品牌栏(居中) → 日期行(左日期/右通知🔴) → (条件)调研待办卡 → 开启之礼进度卡 → **视觉中心：eyebrow"今晨体重"+92px Didot 大数字** → 主按钮"开始称重"+文字链"手动记录" → 隐私微文案 → 近7天 spark → 今日摄入卡(总kcal+四餐格+"拍照打卡") → 饮水/排便/睡眠三指标行(点弹步进器) → "我服务的人"卡 → "今日汇总"卡

**S5 称重流程 `weighing`**（三态）：态A 连接中(脉冲圆环150px+"正在连接"+"模拟失败"ghost) → 态B 读取成功(readout 62px"58.4kg"+"共读取14项"+"保存查看对比") → 态C 手动兜底(下划线输入带昨值+"保存"+"拍秤照")；正式版 15s 超时自动切态C

**S5b 数据对比 `compare`**：返回+标题 → **深绿 hero(墨绿直角+底部ribbon)：三栏(历经天数/体重变化↓/脂肪变化↓ Didot)+一句引文** → cmp-table 14 行 → "值得表扬"卡(3 条✓) → "数值分析"卡(中性解读) → 合规脚注 → "分享到伙伴群"+ghost"回主页"

**S6 拍照打卡 `camera`**（多态）：返回+"AI模拟"角标+标题+文字链"扫营养成分表" → 餐次 portion(早/午/晚/加,按钟点自动选) → 态A 拍照(扫描框190px+隐私提示) → 态B AI结果(food-item列表[名/份量/≈kcal+少标多胶囊+删]+整体反馈卡[左绿条,211三段式]+确认入账[带总kcal]) → 态C 兜底(赭红边框+常见食物库快选)
> **AI 接口契约**（雏形 `pickPhotoAndAnalyze` 已定义，Dev 直接对接）：选图→canvas 压 1280px→`POST {endpoint}/analyze {image:dataURL}`→返回 `items[{name,kcal_est,portion}]`

**S8 今日汇总 `summary`**：返回+标题 → 日期导航(‹前/今天开日历/后›) → 打卡日历 → "今日全维度记录"卡(晨重/摄入/饮水/排便/睡眠) → "今日小结"卡(左绿条) → "老师的话"卡(Didot引文) → **分享长图卡(墨绿直角+ribbon+logo+三数据grid+slogan)** → "转发到微信群"+ghost"保存长图"

## 6. Dev 的 wxml/wxss 转化要点

**✅ 可直接复用**：全部 CSS 变量搬进 `app.wxss`（只取最终值）；flex/grid 布局（小程序支持 `display:grid` 2.7.1+）；发丝线分区；色值/字距/圆角数值。

**⚠️ 需替代**：
| 雏形 | 问题 | 替代 |
|---|---|---|
| `section/div/span/b/i/table/p` | 不存在 | 全换 `view`；`b`→`text`加粗类；`table`→grid 四列；文本必须 `text` |
| inline `onclick` | 无 inline JS | `bindtap`+`data-id`，逻辑进 .js |
| JS `innerHTML` 拼接 renderXxx | 禁 DOM 操作 | **改 setData 驱动 `wx:for`**（最大改造量：雏形拼字符串→小程序数据驱动） |
| inline `<svg>`(丝带/曲线/趋势) | 支持差 | 趋势/曲线用 `canvas`；丝带导出透明 PNG 作背景 |
| `localStorage` | 无 | `wx.setStorageSync`（雏形 key `msm5`/`msm_ai_endpoint`） |
| 内嵌 base64 logo | 臃肿 | 抠成独立 png 放 `/images` 用 `<image>`（3 个 logo 已在 design-reference/assets/） |
| 微信登录/转发 | — | `<button open-type>`、`onShareAppMessage` |

**Didot 字体（关键资产也是最大坑）**：西文衬线中文无此字，fallback 宋体。小程序不能用本地字体，需 `wx.loadFontFace()` 网络字体（要备案域名）或**数字/西文加载 Didot、中文走系统宋体**；务实方案=超大体重数字/分享卡数字做成 canvas 绘制，其余用衬线栈系统匹配；数字务必保 `font-variant-numeric:tabular-nums`。

**迁移优先级**：①app.wxss 落令牌+通用类 ②主线 5 屏(home/weighing/compare/camera/summary)setData 重写 ③cmp-table grid 四列组件化(S5b+S16 复用) ④canvas 处理趋势/曲线,PNG 处理丝带+logo ⑤AI 接口按契约对接。
